# Software Requirements Specification
## QuizMaster — Flutter Mobile App (MVP)

**Version:** 1.0  
**Date:** 2026-04-19  
**Status:** Draft

---

## 1. Introduction

### 1.1 Purpose
This document defines the requirements for the MVP of QuizMaster, a Flutter mobile application that lets students create, organise, and take multiple-choice quizzes generated by AI tools (ChatGPT, Claude, etc.).

### 1.2 Scope
The app covers course management, quiz taking, mistake tracking, and basic progress stats. It runs on Android and iOS as a standalone offline-first app.

### 1.3 Out of Scope (Post-MVP)
- User accounts / cloud sync
- Notifications / reminders
- Quiz shuffle / timer settings
- Confetti animations
- Sharing quizzes with others

---

## 2. Users

**Primary user:** A student who studies independently and wants to practice with AI-generated quizzes on their phone.

No login required. All data is stored locally on the device.

## 3. Functional Requirements

### 3.1 Dashboard

| ID | Requirement |
|----|-------------|
| F-01 | Display total quizzes taken, average score, day streak, and total mistakes as stat cards. |
| F-02 | List all courses as cards showing name, icon, quiz count, and average score badge. |
| F-03 | Provide a search bar to filter courses by name. |
| F-04 | Provide a "New Course" button. |
| F-05 | Tapping a course card navigates to the Course View. |

### 3.2 Course Management

| ID | Requirement |
|----|-------------|
| F-06 | Create a course with: name (required), description (optional), color (picker), icon (emoji picker). |
| F-07 | Edit a course name and description. |
| F-08 | Delete a course with a confirmation dialog; deletes all linked quizzes, results, and mistakes. |
| F-09 | Course View displays: banner with icon/name/description, 4 stats (quizzes, avg score, best score, mistakes), quiz list, and action buttons. |
| F-10 | Action buttons: Add Quiz, Import Quiz (JSON file), Practice Mistakes, Export Course. |

### 3.3 Quiz Management

| ID | Requirement |
|----|-------------|
| F-11 | Add a quiz by pasting a JSON array in a text field, with an optional name. |
| F-12 | Validate the JSON before saving and show a specific error message if invalid. |
| F-13 | Show an in-app AI prompt template the user can copy to use with ChatGPT/Claude. |
| F-14 | Load a sample quiz for first-time users. |
| F-15 | Edit an existing quiz's JSON and name. |
| F-16 | Delete a quiz with a confirmation dialog. |
| F-17 | Export a single quiz as a `.json` file (share sheet). |
| F-18 | Import a quiz from a `.json` file (single quiz or course export format). |
| F-19 | Export a full course (metadata + all quizzes) as a `.json` file. |
| F-20 | Each quiz card in the list shows: name, best score ring, attempt count, last attempt date, and action buttons (Start/Retake, Stats, Edit, Export, Delete). |

### 3.4 Quiz Taking

| ID | Requirement |
|----|-------------|
| F-21 | Display one question at a time with A/B/C/D option tiles. |
| F-22 | Show a sticky header with quiz name and elapsed timer. |
| F-23 | Show a progress bar and "Question X of Y" label. |
| F-24 | Allow navigating between questions via Previous / Next buttons and a dot navigator strip. |
| F-25 | Highlight the selected option visually. |
| F-26 | **Instant Feedback toggle** (Study mode): when ON, immediately show correct/wrong state after tapping an option, lock the answer, and show an explanation banner. |
| F-27 | The Feedback toggle persists as a user preference between sessions. |
| F-28 | "Review Answers" bottom sheet shows all questions with answered/unanswered status; tapping an item jumps to that question. |
| F-29 | "Submit Quiz" prompts confirmation if there are unanswered questions. |
| F-30 | Pausing navigates back to the course; in-progress session is discarded (no resume in MVP). |

### 3.5 Quiz Results

| ID | Requirement |
|----|-------------|
| F-31 | Display a score circle with fraction (e.g. 8/10) and percentage, colored by score range (green / orange / red). |
| F-32 | Show a 4-stat strip: correct, wrong, time taken, change vs. personal best. |
| F-33 | List every wrong answer with the full option list, highlighting the correct answer (green) and the user's wrong answer (red), plus the explanation if available. |
| F-34 | Collapsible "Review All Answers" section showing every question with the same option breakdown. |
| F-35 | Action buttons: Retry Quiz, Practice These Mistakes (visible only if mistakes > 0), Back to Course. |
| F-36 | Show a celebratory animation for a perfect score (100%). |

### 3.6 Mistake Tracking

| ID | Requirement |
|----|-------------|
| F-37 | Automatically save each wrong answer (question + options + correct answer + explanation) linked to its course. |
| F-38 | "Practice Mistakes" on the Course View starts a quiz using all saved mistakes for that course. |
| F-39 | Correct answers during a mistakes-practice session remove the question from the mistakes store. |

### 3.7 Settings

| ID | Requirement |
|----|-------------|
| F-40 | Toggle between Dark and Light theme; persist the preference. |
| F-41 | Export a full backup of all data as a `.json` file. |
| F-42 | Import a backup from a `.json` file (restores all courses, quizzes, results, mistakes). |
| F-43 | "Clear All Data" action with a double-confirmation dialog. |
| F-44 | Display data stats: course count, quiz count, total attempts, data size. |

## 4. Non-Functional Requirements

| ID | Requirement |
|----|-------------|
| NF-01 | **Offline-first.** All features work without an internet connection. |
| NF-02 | **Local storage.** Data persists using SQLite (via `sqflite`) or Hive. |
| NF-03 | **Performance.** Screen transitions complete in < 300 ms on mid-range devices. |
| NF-04 | **Platform.** Supports Android 8+ and iOS 14+. |
| NF-05 | **Responsive.** UI adapts to small phones (360 dp) and large tablets (600+ dp). |
| NF-06 | **Accessibility.** All interactive elements meet a minimum tap target of 44 × 44 dp. |

## 5. Data Models

### Course
```
id          String   (UUID)
name        String
description String?
color       String   (hex)
icon        String   (emoji)
createdAt   DateTime
```

### Quiz
```
id          String   (UUID)
courseId    String   (FK → Course)
name        String
jsonData    List<Question>
createdAt   DateTime
```

### Question (embedded in Quiz.jsonData)
```
question    String
options     List<String>   (2–6 items)
correct     int            (0-based index)
explanation String?
```

### QuizResult
```
id             String   (UUID)
courseId       String
quizId         String?
quizName       String
score          int
totalQuestions int
percentage     int
timeTaken      int      (seconds)
answers        List<AnswerRecord>
completedAt    DateTime
```

### AnswerRecord (embedded in QuizResult)
```
questionIndex  int
selectedOption int
correctOption  int
isCorrect      bool
```

### Mistake
```
id             String   (UUID)
courseId       String
quizId         String?
question       Question (embedded)
incorrectAnswer int
correctAnswer   int
savedAt        DateTime
```

### Settings
```
theme           String   ('dark' | 'light')
instantFeedback bool
```

## 7. Tech Stack (Recommended)

| Concern | Package |
|---------|---------|
| State management | `flutter_bloc` or `riverpod` |
| Local database | `hive` + `hive_flutter` (or `sqflite`) |
| File import/export | `file_picker`, `share_plus` |
| Navigation | `go_router` |
| JSON serialization | `json_serializable` + `freezed` |
| Theme | `flex_color_scheme` |
| Animations | Built-in `AnimationController` + `lottie` (results screen) |

## 8. MVP Scope Summary

**In MVP:**
- Full course + quiz CRUD
- JSON paste + file import/export
- Quiz taking with instant feedback toggle
- Results with wrong-answer breakdown
- Mistake tracking + practice
- Dark / light theme
- Full backup / restore

**Deferred to v2:**
- Cloud sync / user accounts
- Push notifications / study reminders
- Quiz shuffle / time limits
- Social sharing
- Rich text / image questions
